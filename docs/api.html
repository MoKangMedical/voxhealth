API 概览

VoiceHealth API 是一套 RESTful 接口，支持通过语音样本进行健康风险评估。所有端点均返回 JSON 格式响应，使用标准 HTTP 状态码，并通过 Bearer Token 进行身份认证。

API 遵循 REST 规范，支持 JSON 请求体和响应体。所有时间戳使用 ISO 8601 格式 (UTC)。

认证方式

所有 API 请求必须在 HTTP Header 中携带 API Key 进行身份认证：

HTTP Header
Authorization: Bearer vh_live_xxxxxxxxxxxxxxxxxxxx
Content-Type: application/json

获取 API Key

登录 VoiceHealth Dashboard，在 "设置 > API 密钥" 中生成。密钥前缀区分环境：vh_live_ 为生产环境，vh_test_ 为测试环境。

安全提醒

请勿在客户端代码或公开仓库中暴露 API Key。建议通过后端代理转发请求。如发现密钥泄露，请立即在 Dashboard 中轮换。

健康分析

POST /analysis/health 上传语音样本进行健康分析

请求参数

参数类型说明
audio 必填file语音文件，支持 WAV/MP3/FLAC，最大 10MB，建议 16kHz 采样率
languagestring语言代码，默认 zh-CN
metricsstring[]指定分析维度，如 ["respiratory","cardiovascular"]，不填则全量分析
callback_urlstring异步回调 URL，分析完成后 POST 结果至此地址

请求示例

cURL
curl -X POST https://api.voicehealth.mokangmedical.com/v1/analysis/health \
  -H "Authorization: Bearer vh_live_xxxx" \
  -F "audio=@recording.wav" \
  -F "language=zh-CN" \
  -F 'metrics=["respiratory","cardiovascular","mental"]'

批量分析

POST /analysis/batch 批量提交多个语音样本
参数类型说明
samples 必填array语音样本数组，每项包含 audio_url 和可选 metadata
callback_urlstring批量完成后的回调地址
prioritystringnormal | high，高优先级队列加收 50% 费用

单次最多提交 100 个样本。响应立即返回 batch_id，通过 GET /analysis/batch/{batch_id} 轮询进度。

GET /analysis/batch/{batch_id} 查询批量任务状态

参数类型说明

限流策略

API 采用令牌桶限流算法，不同套餐对应不同的请求配额：

套餐请求频率并发数批量上限
Basic10 次/分钟210 样本/批
Pro60 次/分钟10100 样本/批
Enterprise自定义自定义自定义

超限时返回 429 Too Many Requests，响应头中包含：

Response Headers
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1714300800
Retry-After: 45

错误码

所有错误响应遵循统一格式：

错误响应格式
{
  "error": {
    "code": "INVALID_AUDIO",
    "message": "Audio file format not supported",
    "details": "Accepted formats: WAV, MP3, FLAC"
  }
}
  • 400BAD_REQUEST — 请求参数缺失或格式错误
  • 401UNAUTHORIZED — API Key 缺失或无效
  • 403FORBIDDEN — 无权访问该资源，或套餐不支持该功能
  • 404NOT_FOUND — 资源不存在
  • 413PAYLOAD_TOO_LARGE — 音频文件超过 10MB 限制
  • 415UNSUPPORTED_MEDIA_TYPE — 音频格式不支持
  • 422INVALID_AUDIO — 音频质量过低或时长不足 5 秒
  • 429RATE_LIMITED — 请求频率超限，请稍后重试
  • 500INTERNAL_ERROR — 服务器内部错误，请联系技术支持
  • 503SERVICE_UNAVAILABLE — 服务暂时不可用，通常出现在维护窗口

SDK 代码示例

Python

Python
import requests

API_KEY = "vh_live_xxxxxxxxxxxx"
BASE_URL = "https://api.voicehealth.mokangmedical.com/v1"

def analyze_health(audio_path, metrics=None):
    """上传语音文件进行健康分析"""
    headers = {"Authorization": f"Bearer {API_KEY}"}

    with open(audio_path, "rb") as f:
        files = {"audio": f}
        data = {"language": "zh-CN"}
        if metrics:
            data["metrics"] = ",".join(metrics)

        resp = requests.post(
            f"{BASE_URL}/analysis/health",
            headers=headers, files=files, data=data
        )

    resp.raise_for_status()
    return resp.json()

# 使用示例
result = analyze_health(
    "recording.wav",
    metrics=["respiratory", "cardiovascular"]
)
print(f"Respiratory risk: {result['results']['respiratory']['risk_level']}")
print(f"Analysis time: {result['duration_ms']}ms")

JavaScript (Node.js)

JavaScript
const fs = require('fs');
const FormData = require('form-data');
const fetch = require('node-fetch');

const API_KEY = 'vh_live_xxxxxxxxxxxx';
const BASE_URL = 'https://api.voicehealth.mokangmedical.com/v1';

async function analyzeHealth(audioPath, metrics = []) {
  const form = new FormData();
  form.append('audio', fs.createReadStream(audioPath));
  form.append('language', 'zh-CN');
  if (metrics.length) {
    form.append('metrics', JSON.stringify(metrics));
  }

  const resp = await fetch(`${BASE_URL}/analysis/health`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${API_KEY}` },
    body: form,
  });

  if (!resp.ok) throw new Error(`HTTP ${resp.status}`);
  return resp.json();
}

// 使用示例
analyzeHealth('./recording.wav', ['respiratory', 'mental'])
  .then(r => console.log(r.results))
  .catch(console.error);
