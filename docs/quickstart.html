前置条件

开始之前，请确保您已具备以下条件：

  • + Python 3.8+ 或 Node.js 16+
  • + 一段 5-30 秒的语音样本 (WAV/MP3/FLAC)
  • + 有效的 VoiceHealth API Key
1

前往 VoiceHealth 平台注册账户并获取 API 密钥：

3

首次 API 调用

使用以下代码完成第一次语音健康分析：

Python

quickstart.py
from voicehealth import VoiceHealthClient

# 初始化客户端
client = VoiceHealthClient(api_key="vh_live_xxxxxxxxxxxx")

# 上传语音并获取分析结果
result = client.analyze(
    audio_path="sample.wav",
    language="zh-CN",
    metrics=["respiratory", "cardiovascular", "mental"]
)

# 打印结果
print(f"分析ID: {result.analysis_id}")
print(f"耗时: {result.duration_ms}ms")

for metric, data in result.results.items():
    print(f"  {metric}: 风险等级={data.risk_level}, 评分={data.score}")

Node.js

quickstart.js
const { VoiceHealthClient } = require('@voicehealth/sdk');

const client = new VoiceHealthClient('vh_live_xxxxxxxxxxxx');

(async () => {
  const result = await client.analyze({
    audioPath: './sample.wav',
    language: 'zh-CN',
    metrics: ['respiratory', 'cardiovascular', 'mental'],
  });

  console.log(`Analysis ID: ${result.analysisId}`);
  console.log(`Duration: ${result.durationMs}ms`);

  forconst [metric, data] of Object.entries(result.results)) {
    console.log(`  ${metric}: risk=${data.riskLevel}, score=${data.score}`);
  }
})();

4

理解分析结果

API 返回的 JSON 结果包含以下关键字段：

status 分析状态：completed | processing | failed
analysis_id 唯一分析ID，用于后续查询报告
duration_ms 服务端分析耗时 (毫秒)
results.{metric}.risk_level 风险等级：low | medium | high
results.{metric}.score 风险评分：0.0 (最低) ~ 1.0 (最高)

重要提示

VoiceHealth 提供的健康评估仅供参考，不构成医疗诊断。高风险结果应建议用户前往医疗机构进行专业检查。请在您的应用中明确展示此声明。

常见问题

音频文件有哪些格式和时长要求 ▼

支持 WAV、MP3、FLAC 三种格式，文件大小不超过 10MB。音频时长建议 15-30 秒，最短不低于 5 秒。推荐采样率 16kHz 以上以获得最佳分析精度。低于 8kHz 采样率的音频可能影响部分声学特征的提取。

分析结果返回需要多久 ▼

典型的 30 秒音频分析耗时 2-5 秒。Pro 及以上方案享有优先队列，平均响应时间 < 3 秒。如需异步处理，可通过 callback_url 参数指定 Webhook 回调地址，分析完成后系统自动推送结果。

API 返回 401 错误怎么办 ▼

401 表示认证失败。请检查：1) API Key 是否正确，注意区分 test/live 环境；2) Authorization header 格式是否为 Bearer {key}；3) 密钥是否已过期或被轮换。如仍无法解决，请在 Dashboard 中重新生成密钥。

如何在生产环境中安全地存储 API Key ▼

推荐做法：1) 使用环境变量存储密钥，不要硬编码在源码中；2) 在后端服务中调用 API，避免在客户端暴露密钥；3) 使用 CI/CD 的 secrets 管理功能；4) 定期轮换密钥。详细安全指南请参阅 API 文档中的"安全提醒"部分。

能否在本地离线运行 VoiceHealth ▼

可以。Enterprise 方案支持私有化部署，将完整的推理引擎部署在您的内网环境中。需要 GPU 服务器 (推荐 NVIDIA T4 或以上)。私有化版本与云端 API 接口完全兼容，迁移成本为零。具体部署方案请联系销售团队。

分析结果的准确率如何保证 ▼

VoiceHealth 的核心模型在超过 50 万条临床语音数据上训练，整体准确率达到 95% 以上。但准确率受多种因素影响，包括录音环境噪音、麦克风质量、被测者配合度等。我们建议在安静环境下使用专业麦克风采集，并将结果作为辅助参考而非唯一诊断依据。

如何处理大批量的历史录音分析需求 ▼

使用 POST /analysis/batch 端点提交批量任务，Pro 方案单次最多 100 个样本，Enterprise 方案无限制。批量任务支持异步回调和进度查询。如有超大批量需求 (10万+)，请联系我们的技术团队获取专属导入方案。
